# 财务录入与对账测试手册

## 前置
- 启动后端：
  ```bash
  ~/dev/apache-maven-3.9.12/bin/mvn -f backend/pom.xml spring-boot:run
  ```
- 启动前端：
  ```bash
  cd web
  npm run dev -- --host
  ```
- 确认根目录 `.env` 包含：
  - `VITE_API_BASE_URL=http://localhost:8080`

## 用例 A：财务录入公对公款项
1. 使用 `finance@test.com` / `123456` 登录。
2. 进入“对账中心”。
3. 在“录入公对公款项”中填写：
   - 付款方名称
   - 金额
   - 到账日期
   - MG账号（可选）
4. 点击“确认录入”。

预期结果：
- 页面无报错。
- 未匹配列表刷新并显示最新记录（如 MG 账号或支付编号）。

## 用例 B：未匹配对账清单
1. 进入“流水对账”区域。
2. 确认列表显示“待销售关联”的记录。

预期结果：
- 列表展示未匹配付款记录。

## 用例 C：金额不一致异常
1. 查看“对账异常警报”。
2. 若有“金额不匹配”记录，确认显示：
   - 实收金额
   - 合同金额
   - MG账号或支付编号

预期结果：
- 金额不一致异常列表展示正确。

## 常见问题排查
- 无法加载对账数据：确认后端 `/api/reconciliation/unmatched` 与 `/api/reconciliation/mismatch` 可访问。
- 录入失败：确认后端 `/api/finance/bank-transfers` 可访问。
